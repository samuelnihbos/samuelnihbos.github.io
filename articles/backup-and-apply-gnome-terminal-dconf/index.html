How to backup and restore gnome-terminal settings with dconf

20.03.2022 - SamuelNihBos

Dconf is an utility for reading and writing individual values or entire directories from and to a dconf database. I donâ€™t really know what dconf does, but i use it to backup my gnome-terminal settings. Here i will teach you how to backup and apply your .dconf file for gnome-terminal. What you need to do this is just dconf utility (usually called dconf-editor or just dconf).

Install dconf utility

Dconf utility is available on almost all linux distro, including Archlinux, Debian Bullseye, and also Fedora 35. If your distribution is not listed here, you can find dconf package Here.

Exporting gnome-terminal profile settings

First, you need to know your gnome-terminal profile UID, to find out so, run this command dconf list /org/gnome/terminal/legacy/profiles:/. The output will be similar to this

Just copy the very first output to use, it should be the default profile. To actually export your gnome-terminal configuration, run this command. Replace UID with your profile UID!

dconf dump /org/gnome/terminal/legacy/profiles:/:b4d7e88f-8807-48a4-86df-1542d3aae192/ > ~/gnome-terminal.dconf

You can put your dconf file to a safe place like cloud storage or other computers.

Importing your gnome-terminal profile

After exporting your gnome-terminal configuration, you may want to apply that config to other machine. To do so, we need to know the UID of the other machine. To find the UID on the other machine, refer to instruction above. And to apply the configuration, you need to locate the .dconf file. For ease, you can just place the configuration on your $HOME. After locating your .dconf file, run this command. Make sure the UID and the file name correct! dconf load /org/gnome/terminal/legacy/profiles:/:b4d7e88f-8807-48a4-86df-1542d3aae192/ < ~/gnome-terminal.dconf. The changes should apply immediately and you can use your gnome-terminal with the same config!
